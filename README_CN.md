# Agent-Core

> 企业级 AI Agent 引擎。轻量、安全、MCP 原生。Rust 构建。

## 为什么需要 Agent-Core？

### 企业困境

大多数 AI Agent 框架试图包揽一切。内置网页爬取、代码执行、图片生成——什么都有。这在一家企业环境中是**危险的**。

财务人员不需要 Web 编程能力。HR 专员不应该有数据库写入权限。运维人员不应该能执行任意 shell 命令。

**全能型 Agent 不是提升效率，而是安全风险。**

### 我们的思路

Agent-Core 走的是相反的路——它只提供**最基础的 Agent 能力**：

- 对话 LLM
- 调用 MCP 工具
- 执行安全边界
- 管理会话和记忆

其他一切来自**技能**，由管理员分发。Agent 初始是空的。只增长用户需要的那些能力。

```
公司级 MCP（企业通用：认证、HR 政策、组织架构）
    └─ 部门级 MCP（领域技能：财务、人事、运维）
        └─ 项目级 MCP（项目专用工具）
            └─ 用户角色（被分配的技能）
```

**Agent-Core 的目标不是取代人，而是真正的 AI 助手——在职责范围内拥有相应的能力，这才是 Agent 存在的意义。**

## 架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  桌面壳       │────▶│  Agent-Core  │────▶│   Memoria    │
│  (任意 MCP    │     │  (:9753)     │     │  (:9003)     │
│   客户端)     │     │              │     │              │
└──────────────┘     └──────┬───────┘     └──────────────┘
                             │
                     ┌───────▼───────┐
                     │   MCP 源      │
                     │  ┌─────────┐  │
                     │  │ 公司级   │  │  (企业通用)
                     │  ├─────────┤  │
                     │  │ 部门级   │  │  (财务/人事/运维)
                     │  ├─────────┤  │
                     │  │ 项目级   │  │  (项目专属)
                     │  └─────────┘  │
                     └───────────────┘
```

**桌面壳**：任意 MCP 兼容客户端（Jan、Claude Desktop、自研界面）
**Agent-Core**：推理、工具路由、安全、技能管理
**Memoria**：持久记忆、跨 Agent 知识共享
**MCP 源**：定义每个用户/角色能做什么——不多不少

## 设计理念

### 最小核心，最大扩展
Agent-Core 不附带任何领域技能。它是一张白纸。技能从技能市场安装，或由管理员按角色配置。核心保持精简、安全、可审计。

### 安全即默认
7 条红线运行时强制执行：权限递减、代码隔离、治理不可变、数据防泄漏、物理熔断、唯一身份、供应链审查。每次工具调用都经过全部 7 条检查后才能执行。

### 三级 MCP
- **公司级**：企业通用工具（认证、组织架构、合规）
- **部门级**：领域工具（财务、人事、运维、研发）
- **项目级**：项目工具（代码仓库、CI/CD、监控）

### 降级收缩
出问题时 Agent-Core **缩小权限**而非崩溃。MCP 不可用 → 只保留基础查询；LLM 超时 → 自动切换备用 Provider；锁中毒 → 优雅恢复。

### 技能蒸馏
用得越多越聪明。执行日志被蒸馏成可复用的技能模板，重复任务跳过 LLM。

## 功能特性

- **LLM 集成** — 多 Provider 自动 failover
- **MCP 协议** — 原生支持公司/部门/项目三级 MCP 源
- **技能市场** — 按角色安装，白名单控制
- **7 条红线** — 权限链、沙箱、治理、外发、熔断、身份、供应链
- **任务确认** — SAD 风格复述确认
- **技能蒸馏** — 从执行日志自动学习
- **会话管理** — LRU 历史缓存 + 命名空间隔离
- **审计日志** — 结构化审计，敏感字段自动脱敏
- **内置 Web UI** — `http://localhost:9753`

## 配置示例

```toml
# 公司级 MCP
[[mcp_source]]
name = "company-common"
url = "http://company-mcp.internal/common"
token = "${COMPANY_MCP_TOKEN}"

# 部门级 MCP（财务部示例）
[[mcp_source]]
name = "finance"
url = "http://dept-mcp.internal/finance"

# 项目级 MCP
[[mcp_source]]
name = "project-data"
url = "http://localhost:8000/mcp"
```

## 项目结构

```
src/
├── main.rs       — HTTP 服务器、路由、配置
├── agent.rs      — Agent 核心循环、确认状态机
├── llm.rs        — LLM 客户端（流式 + failover）
├── mcp_client.rs — MCP 协议客户端
├── boundary.rs   — 7 条红线
├── composer.rs   — 组合路由
├── approval.rs   — 人工审批
├── audit.rs      — 审计日志
├── session.rs    — 会话状态 + LRU 历史
├── namespace.rs  — 命名空间隔离
├── harness.rs    — 技能蒸馏引擎
└── chat.html     — 内置 Web 聊天界面
```

## 同类对比

| 系统 | vs Agent-Core |
|------|--------------|
| **全能型 Agent** | 内置所有能力，按角色授权困难 |
| **LangChain Agent** | 框架锁定、Python only、无内置安全 |
| **AutoGPT** | 无安全边界、无技能蒸馏 |
| **Agent-Core** | **最小核心 + 按角色赋权、7 条红线、MCP 原生** |
