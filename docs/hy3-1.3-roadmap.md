## 二、技能库（skill library）—— 最自包含，优先落地

**概念**：可复用技能（prompt 模板 / 工具组合 / 子 agent 配方）注册表。agent 按任务语义检索并注入，作为 RoutedLlm 候选池与工具选择的补充能力源。

**当前骨架**（`src/skill_library.rs`）：
- `Skill { id, name, description, trigger_keywords, body, version }`
- `SkillRegistry` trait：`list / get / search_by_task / register / unregister`
- `InMemorySkillRegistry`（关键词重叠打分检索，已带单测）
- 后端路线：`InMemory` → `MemoriaBackedSkillRegistry`（复用 Memoria 存储，存储层就绪后补）

**接入点（后续里程碑）**：
- RoutedLlm 对 Hard 任务的候选池扩展（技能作为「专家配方」参与 Best-of-N）。
- agent 工具选择：检索到的技能注入 system prompt / 工具清单。

## 三、LATS（推理时计算 / Tree search over trajectories）

**概念**：对难任务，展开多条「思考 → 工具 → 观察」轨迹，用 value/judge 选最优路径，替代贪心单步。属 TTC（test-time compute）。

**与现有能力的关系**：
- `RoutedLlm::chat_best_of_n` = **终答** N 选 1（已有）。
- LATS = **过程** N 选 1（新增 planner）。

**设计骨架**：
```
LatsSolver {
  expand(problem)   -> Vec<Node>,            // 展开候选动作
  simulate(node)    -> Outcome,              // 执行/预估
  evaluate(node)    -> f64,                  // value/judge 打分
  search(budget)    -> best_path: Vec<Node>, // 预算/深度上限防爆炸
}
```
- 接入点：RoutedLlm 对 `TaskDifficulty::Hard` 可选 `LatsSolver` 替代单次 `agent.chat` 循环。
- 预算控制：最大深度 / 分支因子 / token 预算，避免指数爆炸。

## 四、HyperAgents（多 agent 组合编排）

**概念**：按子任务动态派生子 agent 并行/串行协作，父 agent 汇总。

**依赖**：技能库 + LATS + 圆桌（圆桌已 native 进 agent-core，commit 56aa8e9）。

**设计**：待 技能库 + LATS 稳定后启动；复用现有 `composer` / `degrade` / `checkpoint` 基础设施。
