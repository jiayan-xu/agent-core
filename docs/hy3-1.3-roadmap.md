## 一、1.3 eval 量化（验收线）

### 1.3.1 分类准确率 ≥90% —— 已闭环，实测 34/36 = 94.4%

- 落地：`src/llm.rs` `eval_tests::classification_accuracy_ge_90_percent`。
- 数据集：**36 条**（13 Easy + 19 Hard 主集 + 2 条多轮）。其中 **2 条为设计内保守误判**：
  - 「服务启动报端口被占用，帮我排查一下」→ heuristic 判 `Easy`（无代码关键词）
  - 「线上数据库连不上，紧急处理」→ heuristic 判 `Easy`
  - 二者计入 `mismatches` 供审视，**不影响** ≥90% 验收线（设计上限即 34/36）。
- **实测真值（2026-07-22）**：`classification accuracy = 34/36 = 94.4%`，测试 `ok`。
  > ⚠️ 三处数字曾不一致：commit 写 33/36、本文旧版写 35/37。**以实测 34/36 为准**。
- **harness 首次运行即抓出真实分类 bug**（叙事价值保留）：「sql 查询最近订单」含「查询」(易白名单) 先命中 → 被压成 Easy；「证明这个定理」缺中文「证明」。已修：难信号优先级 > 易白名单；补「证明」硬信号；长文样本用中性无关键词长句真正覆盖 `len>800`。
- judge 模式分类 eval：待配 `judge_provider` key 后补同类数据集（`classify_by_judge` 已取 last user）。

### 1.3.2 BoN 相对单次 +10pp —— harness 已修，尚未取证

- 落地：`tests/eval_bon.rs`（`#[ignore]`，需 `AGENT_API_KEY` + live LLM + judge key）。
- 机制：同 prompt 跑 `best_of_n=None`（单次）vs `best_of_n=3`（BoN，scorer=Judge）；judge 各打 0–10，统计 `mean_single / mean_bon / Δpp / win-tie-loss`。
- **本轮修复的两处 P1 缺陷**：
  1. **打分 bug**：原 `judge_score` 用 `chars().find_map(to_digit)` 只取**首字符** → judge 回「10」被误判为 1，Δpp 严重失真。改为 `parse_score`：取首个完整数字 token（整数/小数）+ `clamp[0,10]`。
  2. **任务域错配**：原 5 条「开放式创意文案」（slogan/短文/比喻）——战略结论「可验证任务收益大、开放生成有限」。改为 **5 条代码/结构化可验证 prompt**（Python 去重、SQL TopN、TS debounce、二叉树重建、.rs 行数统计），Δpp 更可信。
- **+10pp 是暂定验收目标，非已过关证据**：需 ≥N 条可验样本 + judge 打分稳定后才构成验收；打分 bug 修复前 +10pp 线不可信。
- 运行：`cargo test --release --test eval_bon -- --ignored --nocapture`（默认忽略避免无 key/无网 CI 失败）。
- **🔬 Live 真数（2026-07-22 第一轮，过易任务集）**：`Δpp = 0.0`，`win=0 tie=5 loss=0`，`mean_single=10.00 / mean_bon=10.00`。
  - **根因 = 天花板效应**：5 条代码/SQL/TS 题对 deepseek 过易，judge 对两者均饱和 **10/10**，无方差。**此时 +10pp 不可证真也不可证伪**。
- **🔬 Live 真数（2026-07-22 第二轮，重测：更硬任务 + rubric 细粒度 judge）**：`Δpp = -4.0`，`win=0 tie=4 loss=1`，`mean_single=10.00 / mean_bon=9.60`。
  - 任务集换成**真实质量方差硬题**（线程安全 LRU / IPv4 正则边界 / SQL 修复 / RFC6901 JSON 指针 / bisect_left 溢出）；judge 改**逐条 rubric 扣分（0.5 步长）**避免天花板；生成固定较弱 deepseek-chat 放大方差、judge 用 reasoner、N=3。
  - **结果**：4/5 仍双双 10/10（天花板，无空间）；**1/5（RFC6901）BoN 选择器把 8 分候选当最优、压过单次 10 分** → 净 Δpp = -4.0。
  - **结论（强于第一轮）**：在本配置（N=3 + 本 judge + deepseek-chat）下，BoN **不仅没达 +10pp，还会偶尔回归**。这**证伪了「BoN N=3 默认提分」假设**。
  - **2026-07-22 拍板（E+D）**：「+10pp 硬门」**废除**；BoN 降为**观测指标**（修复 selector 后重测 **Δpp≥0 且不回归**）。三大项开闸不再被 BoN 阻塞。
  - **阶段1 BoN-A（已授权）**：生产 `score_by_judge` 与 eval 对齐（逐候选 `SCORE:` + 相对排序）+ BoN 劣于 temp=0 基线则回退。未过观测门前生产 `best_of_n` 保持关闭。

## 三、LATS（推理时计算 / 过程树搜索）

**概念**：对难任务展开多条「思考 → 工具 → 观察」轨迹，用 value/judge 选最优路径，替代贪心单步。属 TTC。

**与现有能力的关系（纠正旧版接入层误导）**：
- `RoutedLlm::chat_best_of_n` = **终答** N 选 1（已有，BoN）。✅ 旧版此处写对。
- LATS = **过程** N 选 1。⚠️ 旧版写「RoutedLlm 对 Hard 替代 agent.chat 循环」**位置错**：LATS 应挂在 **agent 工具轨迹循环（`execute_chat` 层）**，不是 `RoutedLlm::chat` 终答层。BoN 管终答，LATS 管过程轨迹，二者分层。

**设计骨架**：
```
LatsSolver {
  expand(problem)   -> Vec<Node>,            // 展开候选动作
  simulate(node)    -> Outcome,              // 执行/预估
  evaluate(node)    -> f64,                  // value/judge 打分
  search(budget)    -> best_path: Vec<Node>, // 预算/深度上限防爆炸
}
```
- **正确接入点**：`agent.execute_chat` 的工具调用循环内，对 Hard 任务可插入 `LatsSolver` 替代贪心单步。
- **成本与熔断（DoD 一部分）**：最大深度 / 分支因子 / token 预算（深度×分支×token）；日调用上限；失败/超预算退回贪心（greedy fallback）。

**热路径接线（2026-07-22 完成，默认 OFF）**：
- `lats.rs`：`LatsConfig { enabled, max_depth=3, max_branches=3, daily_token_budget=200000 }`；`LatsController { decide→Search|Greedy / record_tokens / expand_once }`。`expand_once` 用 `LlmClient::chat` 生成 ≤max_branches 候选「下一步」，失败返回空（退回原路径）。带 3 单测（`disabled_is_greedy` / `budget_exhausted_is_greedy` / `enabled_under_budget_searches`）。
- `agent.rs::maybe_lats_expand`：仅 `features.lats=true`（`self.lats=Some`）时展开；`decide()→Search` 时 `expand_once` 注入「## LATS 规划提示」；预算耗尽/Greedy 回 `false` 走原路径。挂在 `execute_chat` 工具注入块后。
- `AgentCore::new`：仅当 `features.lats=true` 才 `Some(LatsController::new(cfg))`。
- `agent.toml [features]` + `[lats]`：`lats = false`，`daily_token_budget` 熔断已内建。
- 当前为**浅层单步展开**；多层回溯 / value 网络留待 G 门复验后深化。

**DoD（当前状态）**：
1. ✅ **单测绿**：`lats` 3 例已绿（含熔断分支）。
2. ✅ **热路径接线**：execute_chat 层、`features.lats` 可控、预算熔断内建。
3. ⛔ **live 冒烟**：Hard 任务轨迹数可观测、预算生效——**未做**（flag 关，G 门未过）。
4. ✅ **可回滚开关**：`features.lats` flag + 预算熔断双重降级。
